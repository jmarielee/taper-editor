# rules-quick-reference.md — one line per rule

`rules.md` is canonical; this file is an index of it. If they disagree, that
disagreement is a bug and `rules.md` wins.

**Scope precondition (gate, not a rule)** — the plan must be (a) AI-generated
and (b) an endurance plan; if either is false, the editor declines to review
and stops.

**Rule 1 — Race-week load (BLOCK)** — for a declared goal race, the final 7
days must show declining load versus the preceding trend, by machine count;
undeclared race priority is a HOLD with one question.

**Rule 2 — Missing ceiling (FLAG)** — every stated limit must define what
happens when it's broken; a limit with no response is decoration.

**Rule 3 — Calendar and arithmetic integrity (FLAG)** — every weekday label
must match a real calendar and every total must recompute, by script, never
by model diligence.

**Rule 4 — Phantom athlete (FLAG)** — every number about the athlete must
trace to something the athlete actually stated or provided.

**Rule 5 — No bad-day rule (FLAG)** — the plan must define behavior for both
the missed session and the added one.

**Rule 6 — Self-certification (FLAG)** — a plan's claims about its own safety
or quality are claims, not facts, and are treated that way.

**Rule 7 — Undeclared race role (FLAG)** — every race in the plan must state
what it's for: the goal, or training for a named larger goal.

**Rule 8 — The solo plan (FLAG)** — the plan must account for the athlete's
total training load, including standing activities outside the plan, or ask.

**Severity** — one block, seven flags: only the check closest to the body
blocks; everything else respects the athlete's authority. Findings use the
four-part contract (THE PLAN SAYS / THE PROBLEM / WHAT IT COSTS / YOUR CALL),
are ranked by consequence, and are capped at five per report.
